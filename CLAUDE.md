# MiniTerminal — guidance for Claude / new sessions

> Read this first when starting a new session. It tells you where the
> project stands and which docs to consult.

## What this project is

A free, open SSH terminal client for iPhone/Android — a no-cost
alternative to Termius. Built with **Flutter** (pure-Dart SSH via
`dartssh2`, terminal rendering via `xterm`).

## Where to look (read in this order)

1. **CURRENT_STATE.md** — single source of truth for "what works now,
   what's next". Always read this before doing anything.
2. **TASKS.md** — backlog / roadmap, grouped by version.
3. **WORKLOG.md** — append-only chronological log. Every session that
   produces a meaningful change appends an entry here.
4. **README.md** — user-facing build & run instructions.

## Working agreement

- Develop on branch `claude/ssh-terminal-app-TGYBf`. Commit + push when a
  stage is complete.
- The remote execution environment has **no Flutter/Dart toolchain**, so
  code cannot be compiled or run here. Write correct code; the user builds
  locally on a Mac for iOS.
- When you finish a stage:
  1. Update **CURRENT_STATE.md** (status, done, next).
  2. Append an entry to **WORKLOG.md** (date, what changed, why, files).
  3. Tick/adjust items in **TASKS.md**.
  4. Commit all of the above together with the code.
- Keep scope tight. Don't add features beyond the agreed version unless
  the user asks.

## Tech decisions (locked)

- Stack: Flutter (chosen by user over native Swift / React Native).
- SSH: `dartssh2`. Terminal: `xterm`.
- Secrets: `flutter_secure_storage` (Keychain/Keystore). Host metadata:
  `shared_preferences`.
- Native `ios/` and `android/` folders are NOT committed; generated by
  `flutter create` locally.
