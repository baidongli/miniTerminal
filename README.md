# MiniTerminal

A lightweight, free SSH terminal client for iPhone/Android — a no-cost
alternative to Termius. Built with Flutter.

**v0.1 scope:** host management (add / edit / delete) + SSH connection with
password auth and an interactive terminal.

## Features

- Save SSH hosts (label, host, port, username)
- Passwords stored in the platform secure store (iOS Keychain / Android
  Keystore), never in plain prefs
- Full interactive terminal (xterm) over a real PTY shell session
- Extra key toolbar: ESC, TAB, CTRL+C/D/L/Z, arrow keys, HOME/END
- Light/dark theme follows the system

## Tech stack

- [`dartssh2`](https://pub.dev/packages/dartssh2) — pure-Dart SSH client
- [`xterm`](https://pub.dev/packages/xterm) — terminal emulator widget
- `shared_preferences` — host metadata
- `flutter_secure_storage` — passwords

## Build & run

This repo contains only the Dart sources and `pubspec.yaml`. The native
iOS/Android scaffolding is generated by Flutter and is not committed.

### Local development (recommended — fastest loop)

With the Flutter SDK installed (Mac for iOS):

```bash
git clone https://github.com/baidongli/miniterminal.git
cd miniterminal
bash tool/setup_local.sh   # generates android/ios, applies the same
                           # patches CI uses, pub get, icons
flutter run                # hot reload on a connected device (press r)
```

`tool/setup_local.sh` mirrors CI exactly so local and CI builds match.
For iOS on a real device a free Apple ID works (7-day, no paid account):
`open ios/Runner.xcworkspace`, set the Signing team once, then
`flutter run -d <ios-device>`.

### Manual setup

1. Install Flutter (3.3+): https://docs.flutter.dev/get-started/install
2. From the project root, generate the platform folders **without**
   overwriting `lib/` or `pubspec.yaml`:

   ```bash
   flutter create --platforms=ios,android --project-name miniterminal .
   ```

   If prompted, keep the existing `pubspec.yaml`/`lib/` files (this command
   only adds the missing `ios/`, `android/`, etc.).
3. Fetch dependencies:

   ```bash
   flutter pub get
   ```
4. Run on a connected device / simulator:

   ```bash
   flutter run
   ```

### No computer? Build in the cloud (Android)

A GitHub Actions workflow (`.github/workflows/android.yml`) builds an
installable APK on every push and attaches it to a rolling pre-release
tagged `android-latest`.

On your Android phone:

1. Open the repo's **Releases** page → "MiniTerminal — latest Android
   build".
2. Download `miniterminal.apk`.
3. When prompted, allow your browser to "install unknown apps", then
   open the APK to install.

The same workflow also runs an **iOS compile check** (no signing) so we
know the code builds for iOS even though installing on an iPhone still
needs a Mac or a paid Apple Developer account.

### Running on a real iPhone

Building/signing for iOS requires a Mac with Xcode:

```bash
open ios/Runner.xcworkspace
```

Set your Apple Developer team under *Signing & Capabilities*, then run
from Xcode or `flutter run` with the device attached. A free Apple ID
works for 7-day sideloaded builds.
